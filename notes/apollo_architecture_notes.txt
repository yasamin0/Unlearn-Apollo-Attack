Apollo attack architecture

1. Inputs
- target unlearned model
- shadow models
- target sample groups: unlearn / retain / test
- sampled evaluation subset size N
- attack configuration, including attack type and weights (for example w)

2. Per-sample signal generation
- for each target sample, Apollo computes two attack signal trajectories:
  - un_conf / un_pred
  - ov_conf / ov_pred
- these signals are generated by the under- and over-based attack procedures
- the summary file stores these detailed per-sample trajectories separately for each group

3. Summary file structure
- the summary pkl contains three top-level groups:
  - unlearn
  - retain
  - test
- each group is stored as a dictionary indexed by sample id
- each sample entry contains:
  - target_input
  - target_label
  - un_conf
  - un_pred
  - ov_conf
  - ov_pred

4. Group-level aggregation
- Apollo aggregates the per-sample signals across:
  - unlearn samples
  - retain samples
  - test samples
- these aggregated signals are then used for threshold-based ternary evaluation

5. Ternary decision mechanism
- Apollo does not rely on a single fixed threshold
- it sweeps over pairs of thresholds
- each threshold pair defines a ternary decision rule
- under each threshold pair, every sample is assigned to one of three classes:
  - unlearn
  - retain
  - test

6. Ternary result objects
- for each threshold pair, Apollo stores:
  - ternary_points
  - threshold_data
  - tpr_results
  - accuracy_results
  - full_classifications
- ternary_points represent the class proportions used for ternary plotting
- threshold_data stores the threshold pairs explored during the sweep
- tpr_results stores class-wise TPR values for unlearn / retain / test
- accuracy_results stores the overall ternary accuracy at each operating point
- full_classifications stores more detailed classification counts and correctness statistics

7. Output files
- summary pkl:
  stores detailed per-sample attack signals for unlearn / retain / test
- ternary pkl:
  stores ternary operating points and threshold sweep results
- ternary all-results pkl:
  stores the full threshold search results including classification statistics

8. Final reported metrics
- best threshold pair
- best ternary accuracy
- TPR for unlearn / retain / test at the best operating point
- empirical one-vs-rest AUC can also be derived from the ternary threshold sweep results

9. Important evaluation note
- the naive AUC computed directly from only the last un_conf / ov_conf values was misleading
- the correct Apollo evaluation should be derived from the ternary threshold sweep results
- therefore the trustworthy final evaluation is based on ternary sweep outputs, not on a naive last-step score extraction

10. Sanity-check findings
- unlearn / retain / test evaluation subsets were mutually disjoint
- no overlap was found among these three sampled groups
- shadow splits also showed zero overlap with the checked target splits in the overlap checks performed so far
- therefore no obvious split leakage was detected from the checks performed up to now
